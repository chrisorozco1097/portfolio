Meter Reading & Usage Tracker

Centralized meter reading tracking system with anomaly detection and trend monitoring.

Technologies

Power BI
Power Automate
MySQL
DAX

Objective

The Utilities Department needed a way to centralize utility meter readings from multiple systems while ensuring data quality before proceeding to billing.
Previously, the process involved disparate spreadsheets and manual checks that resulted in inconsistent tracking and delayed validations.

This project aimed to streamline meter reading ingestion, highlight anomalies, and provide early warning insights, allowing decision-makers to address issues before finalizing reports and charges.

My Role & Techniques

1. Automated ETL with Power Automate

  • Designed an automated flow that retrieves daily meter readings from the Aclara system via CSV ingestion.
  • Transformed and cleaned the data using Power Automate, organizing it into a structured JSON format suitable for Power BI analysis.

2. MySQL Integration

  • Queried historical and reference data directly from MySQL databases, connecting to the facilities management system (AiM) to cross-reference meter details such as building assignments, meter types, and multipliers.

3. DAX Data Modeling

  • Built DAX calculations to:
    • Detect missing readings
    • Identify zero or abnormal usage patterns
    • Summarize monthly and yearly consumption trends
    • Enable custom metrics based on department preferences

4. Interactive Power BI Dashboard

  • Developed an interactive report with:
    • Multi-month meter reading trends
    • Building-level filtering
    • Anomaly detection color code:
      • Green: Current month’s usage 3 times greater than the previous months usage.
      • Orange: Current month’s usage 3 times less than the previous months usage.
      • Purple: Month usage is <60% of or >140% of the previous 13 month average usage.
      • Yellow: Usage zero for the month.
      • Blue: Usage negative for the month or there is no reading.
    • Dynamic search and date filtering to allow quick lookup and validation of readings
    • Historical view across months, along with building names, long descriptions, and usage multipliers
Meter Reading & Usage Dashboard with dynamic filters and alert color coding.

Results & Impact

  • Centralized data from multiple utility systems into a single dashboard
  • Allowed the Utilities Director to identify abnormal trends prior to billing cycles
  • Reduced manual data validation time from hours per week to minutes
  • Established a color-coded early warning system to proactively address anomalies
  • Enabled historical tracking across months and years for improved forecasting and reporting

Why It Matters

Utility data accuracy has a direct financial impact on operational costs and budgeting decisions.
This project delivered a single source of truth for utility usage while equipping leadership with real-time insights to prevent billing errors.

By flagging inconsistencies upfront, the department can investigate operational issues, prevent revenue leakage, and ensure accurate internal and external billing.

The combination of automated workflows, clean data pipelines, and interactive visualizations makes the system sustainable and scalable for future expansion, such as adding water, gas, or chilled water metering.
